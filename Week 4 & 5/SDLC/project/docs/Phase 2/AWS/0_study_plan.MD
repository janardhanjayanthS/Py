## Phase 1 ‚Äî AWS Fundamentals & IAM
**Duration:** 1‚Äì2 weeks

### Goals
Understand how AWS works as a platform before touching any specific service.

### Topics
- [ ] How AWS accounts, regions, and availability zones work
- [ ] The AWS Console, CLI, and SDK basics *(pick one: Python/Boto3 or Node.js)*
- [ ] Billing basics and the AWS free tier
- [ ] The shared responsibility model

### IAM ‚Äî Do This First, Always
IAM is the backbone of every AWS service. Nothing works securely without it.

- [ ] Users, groups, and roles
- [ ] Policies ‚Äî inline vs. managed
- [ ] Least privilege principle
- [ ] Identity-based vs. resource-based policies
- [ ] How services assume roles to talk to each other

### üì∫ Videos
- [AWS IAM Tutorial for Beginners ‚Äî freeCodeCamp YouTube](https://www.youtube.com/results?search_query=AWS+IAM+tutorial+freeCodeCamp) ‚Äî free, complete beginner walkthrough
- [AWS IAM Deep Dive ‚Äî Abhishek Veeramalla (Day 2, Zero to Hero)](https://www.youtube.com/@AbhishekVeeramalla) ‚Äî practical with real demos
- [AWS re:Invent ‚Äî IAM Best Practices](https://www.youtube.com/results?search_query=AWS+reinvent+IAM+best+practices) ‚Äî watch at least one re:Invent IAM talk

### üìñ Docs & Reading
- [IAM Getting Started ‚Äî AWS Docs](https://docs.aws.amazon.com/IAM/latest/UserGuide/getting-started.html)
- [IAM Best Practices ‚Äî AWS Docs](https://docs.aws.amazon.com/IAM/latest/UserGuide/best-practices.html)
- [IAM Policies Deep Dive ‚Äî AWS Fundamentals Blog](https://awsfundamentals.com/blog) *(search "IAM policies")*

### Practice
- Create an AWS free-tier account
- Set up the AWS CLI with a non-root IAM user
- Write a policy that grants S3 read-only access and attach it to a test user

### Key Question From This Point Forward
> *"How does IAM control access to this service?"* ‚Äî ask this for every new service.

## üß† Core Mindset Rules

1. **Always ask:** *"How does IAM control access to this?"* ‚Äî for every new service.
2. **Always ask:** *"How do I see what went wrong?"* ‚Äî CloudWatch logs and X-Ray traces.
3. **Build things, break them, read the logs.** That is the fastest way through this list.
4. **Use the free tier aggressively.** Almost everything here has a free tier allowance.
5. **Deploy with code from day one.** Get comfortable with SAM early ‚Äî avoid clicking through the console for everything.
6. **Delete resources when done.** Always clean up to avoid surprise bills.
